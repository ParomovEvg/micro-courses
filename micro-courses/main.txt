micro-courses 

Идея проекта

Создать глобальную иерархическую систему курсов

Есть множество институтов

В институте есть курсы

Институт
====
Институт это сущность верхнего уровня

Институт является областью видимости для прав управления, курсами института
Институт создаётся пользователем, который получает все права на институт
Он может создавать роли, назначать пользователей на эти роли.

Курсы
====

Тело курса

Курсы это древовидная структура, узлы которой служат для структуризации знаний
Создатель курса может создавать любой вложенности структуру. Листьями этого дерева являются этапы.

Этап состоит из виджетов. Виджет взаимодействует с этапом по средствам специального api

Все параметры этапа определяются виджетами которые находятся в этом этапе. 

Виджеты разрабатываются нами или сторонними разработчиками, расостранятются по средствам маркетплейса
Могут быть платными или бесплатными. По сути виджет это приложение, которое имеет своё состояние. 

Виджет должен реализовать несколько интерфейсов.
Интерфейс создания\редактирования

Интерфейс прохождения

Интерфейс проверки

Так же виджеты могут предоставлять api для сбора статистики, и обработки данных
Соответствующих специальному интерфейсу

Пример виджетов - Лекция, тест, сендбокс с кодом

=====
Почему виджеты такие

Виджеты должны быть основной осью роста функциональности приложения

Для создателя курса есть несколько путей
- Использование \ Покупка виджетов из маркетплейса
- Создание собственного виджета
- Модификация готового виджета

Таким образом при достижении определённого размера, функциональность должна начать расти за счёт комьюнити.

=====
Древовидность курсов

У курса могут быть критерии доступности
1. Курс может требовать прохождения курса из этого института
2. Курс может требовать прохождения курса из дрогого института
3. Курс может требовать вступительного экзамена

На основе этих требований строится карта курсов института

Приложение сможет анализировать эти структуры и составлять дорожные карты для студентов
